## Interview Prep Notes

### AI Engineer vs Data Scientist Interviews

| Dimension | Data Scientist | AI Engineer |
|---|---|---|
| Primary output | Insights, models, reports | Deployed AI products and systems |
| Core language | Python (notebooks) | Python or TypeScript (production code) |
| LLM APIs | Optional | Core requirement |
| RAG systems | Rarely required | Standard expectation |
| Agent frameworks | Rarely required | LangChain, LlamaIndex, or custom |
| Production deployment | Occasional | Non-negotiable |
| System design | Rarely tested | Standard interview component |

### Typical Interview Format

1. **Recruiter Screen** (30 min) — background, interest in AI
2. **Technical Phone Screen** (45-60 min) — LLM concepts, may include live coding
3. **System Design for LLM Systems** (45-60 min) — e.g., "Design an AI document Q&A system"
   - Chunking strategy, embedding model selection, vector index design
   - Query routing, reranking, latency budget, cost estimation
   - Failure modes and monitoring
4. **Take-Home Project** (48-72 hrs) — build a working agent or RAG pipeline
5. **Behavioral / Presentation** (30-60 min) — walk through projects, past failures

**Note:** AI eng interviews typically do NOT do heavy LeetCode. Focus is on system design and practical AI building.

### Resume Reframing Tips

| Old Framing | New Framing |
|---|---|
| "Built ML pipeline for X" | "Designed and deployed production AI system handling X" |
| "Implemented RAG prototype" | "Built RAG system with [vector DB], [embedding model], [chunking strategy] serving N queries/day" |
| "Explored LLM use cases" | "Shipped LLM-powered feature reducing [metric] by X%" |
| "Built data pipeline" | "Engineered data infrastructure supporting real-time AI inference" |
